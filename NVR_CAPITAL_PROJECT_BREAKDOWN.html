Executive Summary

Never Rest Capital (NVR) is an autonomous trading agent operating 24/7 on Base L2. It manages real capital — analyzing markets, executing trades via DEX aggregators, harvesting profits, and continuously self-improving its own strategy thresholds based on realized performance.

Over the course of this refactoring initiative, the codebase has been transformed from a single 17,451-line monolith into a modular, type-safe, tested architecture ready for multi-user deployment.

Remaining Steps to Production Scale

Priority 1: Merge & Deploy BLOCKING

The refactoring branch has 40+ commits of pure architectural improvement with zero behavioral changes. Merge to main and redeploy. The bot trades identically — just with a vastly better codebase underneath.

Priority 2: Centralize State Fully HIGH VALUE

The state store exists at src/state/store.ts. Two modules (diagnostics, portfolio) already import directly. Next: migrate src/server/ and src/dashboard/ to use the store instead of context injection. This eliminates the 120+ field ServerContext interface.

Priority 3: Telegram Phase 2 — Command Handlers HIGH VALUE

Currently Telegram is alert-only (one-way). Adding command handlers enables users to:

  • Check portfolio status: /status, /portfolio, /pnl
  • Manage positions: /sell TOKEN, /harvest
  • Monitor health: /health, /errors
  • Control the bot: /pause, /resume

This is critical for scaling — users need to interact with their bot without a dashboard.

Priority 4: Expand Test Coverage ONGOING

90 tests cover algorithm, portfolio, capital, and self-improvement. Next targets:

  • Server route handlers (API response shapes)
  • Trade execution (mock RPC, verify calldata)
  • State persistence (save/load cycle integrity)
  • Integration tests (full trading cycle with mocked market data)

Priority 5: Move Toward strict: true Everywhere ONGOING

12 of 15 src/ modules already pass strict. Remaining:

  • src/server/ — complex context types need refactoring
  • src/dashboard/ — similar context pattern
  • agent-v3.2.ts — long-term goal, requires incremental cleanup

Priority 6: Services Cleanup ONGOING

  • Remove unused trade-queue.ts (168 lines, nothing imports it)
  • Consider splitting derivatives-strategy.ts (926 lines) if it grows
  • Group equity services (alpaca-client, stock-data, equity-strategy)
  • Group yield services (aave-yield, morpho-yield, yield-optimizer)

Priority 7: Multi-User Infrastructure SCALING

For hundreds of users, each running their own bot instance:

  • Dockerized deployment with env-based configuration
  • Per-user state isolation (separate state.json per instance)
  • Centralized monitoring dashboard (aggregate health across fleet)
  • Onboarding flow: wallet setup, initial deposit, risk profile
  • Rate limiting on shared API keys (CoinGecko, FRED, etc.)
